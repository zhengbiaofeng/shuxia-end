# Design System: 书匣后台管理

## 1. Visual Theme

书匣后台是一个 NAS 私有内容库管理界面。视觉语言应当冷静、清晰、低噪，接近专业控制台而不是营销页面。界面密度为中高密度：信息量足够管理长期使用，但每个区块之间要有明确留白和边界。

- **Density:** 7/10，适合表格、筛选、指标卡和配置弹窗。
- **Variance:** 3/10，后台页面以稳定、可预测、对齐清楚为主。
- **Motion:** 2/10，动效只用于 hover、focus、弹窗进入和加载反馈。
- **Core Principle:** 视觉稿服务外观，交互规范优先于视觉稿漏洞。

## 2. Color Palette

- **Admin Canvas** `#f5f7fb`：页面背景。
- **Surface White** `#ffffff`：卡片、表格、弹窗主体。
- **Title Navy** `#071f56`：页面标题、弹窗标题、强强调文本。
- **Body Navy** `#172554`：正文主文本、表单文本。
- **Muted Blue Gray** `#50679b`：说明、辅助文案、表格次级信息。
- **Quiet Border** `#dbe5f4`：输入框、按钮、弹窗内部结构线。
- **Panel Border** `#e7ecf7`：卡片和表格容器边框。
- **Primary Blue** `#1476ff`：主按钮、激活态、链接、进度条、焦点态。
- **Success Green** `#16a34a`：正常状态、成功状态。
- **Warning Orange** `#f97316`：连接异常、警告状态。
- **Danger Red** `#ff3b4f`：删除、危险操作。

禁止使用纯黑 `#000000`、高饱和霓虹色、大面积紫蓝渐变、随机暖灰/冷灰混用。

## 3. Typography

字体栈：

```css
'Noto Sans SC', 'Microsoft YaHei', system-ui, sans-serif
```

固定字号规范：

- **辅助说明 / badge / table meta:** `12px`
- **表格正文 / 菜单子项 / 页脚:** `13px`
- **表单 label / 输入框 / tab / 按钮 / 正文:** `14px`
- **分组标题 / 卡片标题:** `15px` or `16px`
- **弹窗标题:** `20px`
- **页面标题 / 内部模块标题:** `21px`
- **软件名称 / 品牌标题:** `24px`
- **指标数字:** `26px`

字重规范：

- 普通正文：`400`
- 表单、tab、菜单：`700`
- 软件名称、页面标题、弹窗标题、关键数字：`800`

层级规范：

- 软件名称必须高于内部模块名称，品牌标题默认 `24px / 800`。
- 页面模块标题默认 `21px / 800`，不得大于软件名称。
- 侧边栏品牌副标题保持 `12px`，作为辅助信息，不参与主标题竞争。

后台界面不要使用 serif 字体、花体字、负字距或 viewport 线性缩放字号。页面内容应优先使用固定语义 token，而不是随意写新的 clamp。

## 4. Layout Rules

页面：

- 侧边栏宽度：`218px`
- AppShell 只负责 sidebar + main 布局，禁止给内容区重复加 padding。
- 主内容内边距：桌面 `24px 24px 14px`，移动端 `20px 16px`
- 页面标题区和 tab 区间距：`27px`
- 指标卡和主要内容区间距：`22px-24px`
- 卡片栅格间距：`16px`
- 表格容器圆角：`8px`
- 表格行高：`56px-72px`

表格：

- 表头背景：`#fbfcff`
- 表头字号：`13px / 800`
- 单元格字号：`13px`
- 行分割线：`#e9eef8`
- 操作列优先使用图标按钮。

响应式：

- 小于 `980px` 时，页面顶部、操作区、表格 footer 改为纵向布局。
- 小于 `760px` 时，弹窗、表单、卡片列表降为单列。
- 不允许页面级横向滚动；表格自身可以在容器内横向滚动。

## 5. Component Rules

### Buttons

- 主按钮高度：`40px`
- 主按钮最小宽度：`96px`
- 主按钮背景：`#1476ff`
- 次按钮白底描边，边框 `#dbe5f4`
- 圆角统一 `6px`
- active 状态允许轻微 `translateY(1px)`，禁止外发光。

### Cards

- 卡片圆角：`8px`
- 卡片边框：`#e7ecf7`
- 卡片阴影：轻量、蓝灰色调，不使用黑色厚阴影。
- 卡片只用于指标、表格容器、分类树、弹窗，不做卡片套卡片。
- 指标卡统一采用内容管理卡片样式：顶部为 `36px` 彩色图标 + 标题，中部为大数字，底部为辅助信息。
- 指标卡必须优先复用 `src/components/common/MetricCard.vue`；资源库、内容管理、概览页不得各自发明不同指标卡结构。

### Inputs

- 输入框高度：`39px-40px`
- 输入框圆角：`6px`
- 输入框边框：`#dbe5f4`
- placeholder：`#8394b7`
- label 左侧固定列宽，复杂弹窗默认 `136px`，SMB 类紧凑表单可用 `112px`
- helper text 放在输入框下方，字号 `13px`

### Icons

- 使用 Element Plus icons。
- 一个页面内不要混用多套图标库。
- 侧边栏图标必须来自同一个菜单源，避免不同页面激活时图标风格变化。

## 6. Modal Rules

弹窗是后台配置的核心组件，必须优先遵守这套规范。

默认弹窗采用 **方案 A：标准三段式弹窗**，参考图：`UI/modal-proposals/modal-proposal-a-classic.png`。

尺寸：

- `sm`: `520px`，确认和简单表单。
- `md`: `640px`，普通编辑表单。
- `lg`: `784px`，复杂配置表单，例如添加存储。
- 移动端：`calc(100vw - 28px)`。
- 同一个弹窗流程内，宽度禁止随 tab、类型、步骤切换而变化。

结构：

- 遮罩：`rgba(19, 31, 55, 0.48)`，可加 `blur(2px)`。
- 弹窗圆角：`8px`
- Header：`24px 32px 18px`，底色 `#f6faff`。
- Body：`22px 32px 24px`，底色 `#ffffff`。
- Footer：`18px 32px 20px`，底色 `#f4f8fe`。
- Header / Body / Footer 之间必须用 `#dbe5f4` 结构线区分。
- 标题：`20px / 800`
- 副标题：`14px / 600`
- 分组标题：`15px / 800`，左侧 `3px` 蓝色竖线。
- Footer 按钮右对齐，取消在左，确定在右。
- 内容超高时，只滚动 body，header 和 footer 固定在弹窗内部视觉位置。

交互：

- 弹窗内切换类型、tab、步骤时，必须保留返回或继续切换的入口。
- 如果设计图让用户陷入单向状态，应修正设计稿，不盲目还原。
- 关闭方式包括右上角关闭、取消按钮、必要时支持点击遮罩。

## 7. Storage Modal Rules

添加存储弹窗使用 `lg` 宽度，四种类型共享同一个弹窗容器：

- 本地目录
- SMB
- WebDAV
- 网盘

要求：

- 存储类型卡片必须常驻，不能只在本地目录状态显示。
- 切换类型时弹窗宽度不变。
- 网盘类型可额外显示「选择网盘类型」区。
- 连接设置、基本设置、高级选项都使用相同分组标题样式。
- 测试连接和授权方式属于表单内交互，不应改变弹窗结构宽度。

## 8. Anti-Patterns

禁止以下行为：

- 弹窗切换内容时宽度忽大忽小。
- 设计稿中存在交互漏洞时仍机械还原。
- 同一个侧边栏在不同页面使用不同菜单数据源或不同图标风格。
- 每个页面临时发明字号、圆角、按钮高度。
- 卡片套卡片、区块套大卡片。
- 使用大面积渐变、发光、装饰圆球。
- 使用 emoji 作为后台功能图标。
- 表单没有 helper/error 位置。
- 重要操作只靠颜色区分，缺少文字或图标语义。

## 9. Implementation Notes

代码优先使用 `src/styles/tokens.css` 中的语义 token。若某个组件出现大量硬编码颜色、字号、圆角，应先判断是否需要新增 token，而不是继续复制数值。

新增后台页面时，优先复用：

- `ResourceShell`
- `ResourceMetricGrid`
- `ResourcePagination`
- `StorageTable` / `TagTable` / `CategoryBoard`
- `AddStorageModal`

新增弹窗时，优先遵守 `Modal Rules`，并把不同状态做成配置或插槽，避免拆成多个互相漂移的弹窗实现。
